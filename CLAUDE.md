# CLAUDE.md - このプロジェクトの作業指針

## Git 管理について

- **コミット・push はユーザーが明示的に指示したときのみ行う**。勝手にコミットしない。
- ファイルを変更したあとは「コミットしますか？」と確認するのではなく、ユーザーの指示を待つ。
- Git の設定・操作（remote, branch, tag 等）はこちらで担当してよい。

## 対応環境

このプロジェクトは以下の環境すべてで動作することを目指す。スクリプトや設定は特定の環境に依存しないよう汎用的に作ること。

| 環境 | GPU | VRAM | CUDA Arch |
|------|-----|------|-----------|
| 開発（WSL2） | RTX 4070 Ti | 12GB | sm_89 |
| 本番A | RTX 3090 Ti | 24GB | sm_86 |
| 本番B | A100 40GB | 40GB | sm_80 |
| CPU only | なし | - | - |

- ビルドスクリプトは `CUDA_ARCH` 環境変数で切り替え可能にし、デフォルトは `native`（自動検出）とする。
- CPU only でも動作できるよう、GPU を前提とした実装は避ける。

## アーキテクチャ方針

- **llama-server**（llama.cpp 組み込みサーバー）を推論バックエンドとして使う。
- Python（llama_cpp ライブラリ）から直接モデルをロードする方式は **使わない**（並列化の制約があるため）。
- **Python サーバー（FastAPI 等）** は llama-server の前段に置き、ルーティング・ロードバランス・認証を担う。
- llama-server は OpenAI 互換 API を提供する（`/v1/chat/completions` 等）。

## 実装方針

- 環境差異を吸収するため、ハードコードは避けて設定ファイル・環境変数で切り替えられるようにする。
- `models/`、`llama.cpp/`、`.env` は Git 管理外。
- スクリプトは WSL2 で開発し、GitHub 経由で別サーバーに展開する運用を前提とする。
- Python パッケージ管理は **uv** を使う。`pip` は使わない。
